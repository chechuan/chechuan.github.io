创建钉钉消息待办

發表於2020-11-19|更新於2023-10-19|第三方应用
|閱讀量:

前言

昨天同学说起一功能，想着有空把它加上，就是，我们的钉钉消息待办，我们可以把它理解为和钉钉机器人一样，首先写个脚本，然后在审批人审批之后，调用该脚本，向用户的钉钉客户端发送一条待办消息。后续工作可以参考官方开发文档https://ding-doc.dingtalk.com/doc#/serverapi2/gpmpiq

准备工作

首先登录钉钉开放平台，如果没有账号可以注册一个。https://open-dev.dingtalk.com/#/index

管理密码，必须由数字、小写字母、大写字母组成，否则提示输入不正确。

  • 第一步创建应用（进入首页之后点击”应用开发”）

    点击确定，创建完成。

    这里，是我创建的应用https://open-dev.dingtalk.com/#/corph5

​ 生成应用凭证一定要保存好。

  • 第二步点击权限管理添加权限

    添加权限

  • 第三步把本机ip添加白名单

创建待办

  • 第一步获取token凭证 https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=/get_jsapi_ticket

  • 第二步获取userid https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=/user/getUseridByUnionid

    • 根据unionid和token获取userid

    • 如果不知道自己的unionid可以根据手机号获取useridhttps://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=/user/get_by_mobile

      如果获取不到,就把返回的报错信息后面的ip添加白名单，再次获取。

  • 第三步创建待办https://open-dev.dingtalk.com/apiExplorer#/?devType=org&api=/workrecord/add

    更新待办

    企业可以调用该接口更新待办事项状态，调用成功后，该待办事项在该用户的“待办事项”列表页面中消失。

    调试工具：在线调试

    请求方式：POST（HTTPS）

    请求地址：https://oapi.dingtalk.com/topapi/workrecord/update?access_token=ACCESS_TOKEN

    参数说明：

    参数 类型 必须 示例值 说明
    userid String 是 manager70 待办事项对应的用户id
    record_id String 是 record123 待办事项唯一id。该id可以使用创建待办接口中传入的biz_id，也可以使用创建待办接口返回结果中的record_id
